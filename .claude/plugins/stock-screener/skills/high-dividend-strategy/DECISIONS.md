# high-dividend-strategy 设计决策

## 1. 核心定位

把高股息策略改写为“高股东回报”策略，核心判断对象是分红质量、分红稳定性和未来回报结构，而不是单一股息率排名。

## 2. 候选池底座

继续使用 `.claude/skills/lixinger-screener` 做批量建池，原因是：
- 先筛出红利候选，比逐只查分红更高效
- 当前仓库已有可运行的红利相关模板
- 能把 OpenAPI 调用控制在入围股范围内

## 3. 历史分红的获取方式

历史分红统一通过 `cn/company/dividend` 补查，原因是：
- 接口语义清晰
- 适合核对公告日期、分红金额、除权日
- 更适合对少量重点公司做深查

## 4. 关键判断框架

红利分析统一拆成四层：
- 分红质量：股息率、分红率、现金流/资产负债支撑
- 分红稳定性：连续性、周期性、波动性
- 分红增强：提分红、特别分红、回购、市值管理线索
- 总回报：股息 + 估值修复 + 盈利稳增

## 5. 输出策略

结果至少拆成：
- `稳定收息`
- `分红成长`
- `央国企分红重估`
- `高股息陷阱预警`

这样可以把“现在能收息”和“未来更有提升空间”区分开。

## 6. 数据边界

当前仓库最可靠的是：
- 候选池初筛
- 历史分红补查
- 基本估值与财报补查

当前不做的事情：
- 用单一股息率代替长期回报判断
- 在缺少现金流或公告证据时断言分红一定可持续
- 默认构造不存在的输入文件或未验证字段
