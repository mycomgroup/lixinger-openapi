# 北交所策略

围绕“先判断能不能交易，再判断值不值得研究”执行。

## 何时使用

在以下场景使用：
- 用户要找北交所机会或北证50范围内的标的
- 用户关心专精特新、细分龙头、流动性折价
- 用户需要明确仓位、滑点与退出条件

## 执行步骤

### 1. 明确交易约束

优先确认或默认以下参数：
- 范围：`北证50` / `全北交所`
- 数量：默认 10 只
- 流动性门槛：必须明确
- 侧重点：`流动性错杀` / `业绩兑现` / `专精特新`

### 2. 先做候选池，再谈成长

优先使用 `lixinger-screener` 做北交所候选池，详见 [references/data-queries.md](references/data-queries.md)。

如果 request 版字段映射不稳定，可切回 browser 版做字段校验，但最终结论仍要以可复现的数据为准。

### 3. 把可交易性放在第一优先级

具体框架见 [references/methodology.md](references/methodology.md)。

至少从以下角度判断：
- 日均成交额与换手率是否足够支撑交易
- 估值与成长是否匹配
- 是否具备专精特新或细分龙头特征
- 是否存在解禁、客户集中、波动过大等硬风险

### 4. 强制输出异常发现

每次执行都必须输出：
- `本次异常发现`
- `潜在机会清单`
- `未来跟踪信号`

异常发现优先写：
- 业务强但流动性折价严重
- 成长兑现不错，但市场关注度仍低
- 估值便宜不是真便宜，而是流动性或基本面风险的结果

### 5. 做机会分类

最终至少分类为：
- `流动性错杀`
- `专精特新龙头`
- `业绩兑现型`
- `北交所高风险样本`

### 6. 附执行清单

每次输出都要附：
- 仓位建议或仓位上限
- 滑点与成交约束
- 退出触发条件

## 输出要求

按 [references/output-template.md](references/output-template.md) 输出，先给结论和可交易性，再展开成长与估值论证。

## 数据边界

- 北交所策略不能照搬主板逻辑
- 没有流动性约束的结论视为不完整
- 若专精特新、订单或产业属性缺乏可验证数据，必须显式标注“待补充”
