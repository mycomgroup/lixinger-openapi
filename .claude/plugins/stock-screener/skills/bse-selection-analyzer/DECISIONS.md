# bse-selection-analyzer 设计决策

## 1. 核心定位

把北交所策略定义为“流动性折价中的成长机会识别”，而不是简单复制主板的小盘成长筛选。

## 2. 优先级排序

统一采用以下优先级：
1. 可交易性
2. 成长兑现
3. 估值与再评级想象力
4. 专精特新与产业属性

原因是北交所的最大约束往往不是故事，而是流动性与执行成本。

## 3. 候选池与补查分工

- 候选池：优先通过 `lixinger-screener` 快速缩小范围
- 指数样本、行情与财报补查：走 `query_data`

这样可以避免为少量可交易样本付出过大的数据成本。

## 4. 历史行情接口选择

统一放弃不存在的 `cn/company/quote/daily`，改用 `cn/company/candlestick`。这是当前仓库内可验证且可维护的做法。

## 5. 输出策略

结果至少拆成：
- `流动性错杀`
- `专精特新龙头`
- `业绩兑现型`
- `北交所高风险样本`

并且每次都要附：仓位上限、滑点假设、退出条件。

## 6. 数据边界

当前仓库内最可靠的是：
- 北证50成分补查
- 行情与换手验证
- 基本估值与财报补查

当前不做的事情：
- 在缺乏验证数据时硬贴“专精特新龙头”标签
- 省略流动性与执行风险
- 默认给出不可执行的大仓位建议
