# DECISIONS — board-crowding-risk-monitor

## 设计决策记录

### D1：为什么拥挤度评分是四维而不是更多维度

**背景**：拥挤度可以从更多维度衡量（如融资余额变化、北向持仓比例、期权Put/Call比率等）。

**决策**：核心四维：换手率分位、成交占比、估值分位、资金集中度。

**理由**：
1. 四维均有理杏仁 API 的直接或间接支撑，数据可获取性高
2. 过多维度会导致每个维度权重过低（<10%），边际贡献意义不大
3. 四维已覆盖量（换手/成交）+ 价（估值）+ 资金（流向）三个核心维度，分析框架完整
4. 其他维度（如期权数据）在 A 股数据可得性极低，引入会导致频繁降级

---

### D2：为什么设置30/25/25/20这个权重而不是等权

**背景**：四维等权（各25%）是最简单的设计。

**决策**：换手率权重最高（30%），资金集中度权重最低（20%）。

**理由**：
1. **换手率是最直接的拥挤度代理**：大量散户资金涌入必然体现在换手率上，且历史数据丰富，分位数可靠
2. **资金集中度数据可得性最低**：北向+融资数据通常有延迟且对部分板块覆盖不全，权重设低以减少缺数时的评分失真
3. **成交占比和估值分位各25%**：成交占比反映全市场的资金竞争，估值分位提供基本面维度的佐证，两者同等重要

---

### D3：脆弱触发器为什么是"OR 任一触发"而不是"AND 多个同时触发"

**背景**：可以要求多个触发器同时满足才算真正危险。

**决策**：触发器独立判断，单个 HIGH 级触发 + 高拥挤度即触发建议减仓。

**理由**：
1. A股市场波动速度极快，等待"AND"多个触发器同时满足时，往往为时已晚
2. 脆弱触发器每个都有独立的经济含义，单个触发已代表实质性风险信号
3. 每个触发器的严重程度通过 HIGH/MEDIUM 分类体现，而非通过"AND"叠加
4. 与 qc-rules.schema.json 的设计理念一致：宁可早警告，不要晚发现

---

### D4：为什么龙头破位触发器不要求"连续2日"而是"单日"

**背景**：为避免误报，可以要求龙头股连续2日跌破20日均线才触发。

**决策**：单日跌破即触发，但设置为 HIGH 级别（而非最高级）。

**理由**：
1. A股板块行情逆转往往在1-2日内发生，等待连续确认时机会成本过高
2. 龙头股跌破均线本身已是强信号（尤其在高拥挤度背景下）
3. 通过"失效条件"（翌日收回均线以上则解除）来处理假信号问题
4. 将其设为 HIGH 而非 CRITICAL，避免过度恐慌
