# 板块拥挤度监控器

扮演量化风控分析师。通过"四维拥挤度评分 + 五类脆弱触发器识别"框架，为高热板块提供风险刹车——明确指出当前板块是否已过度拥挤，给出降仓/观察/回避的可操作建议。

---

## 工作流程

### 第一步：确认输入参数

确认以下内容：
1. **研究主体**：目标行业名称（如"电子"）或概念板块名称（如"AI算力"）
2. **主体类型**：`industry`（行业板块）/ `concept`（概念板块）
3. **告警阈值**：拥挤度评分告警阈值，默认 80（>80 为高拥挤）
4. **分析基准日期**：默认今日

### 第二步：数据获取

从 `references/data-queries.md` 获取：
- 行业换手率（`to_r`）及其历史分位数数据
- 行业成交额（`ta`）及全市场成交总额（用于计算占比）
- 行业 PE 10年历史分位数（`pe_ttm.y10.mcw.cvpos`）
- 龙头股价格数据（用于龙头破位检验）

概念板块（`type=concept`）：
- 通过 AKShare `stock_fund_flow_concept` 获取资金净流入数据
- 通过涨停家数 + 换手率近似计算热度

若数据部分缺失，在 `data_gaps` 中标注，降级为三指标简版（换手率+成交+估值）。

### 第三步：四维拥挤度评分（0-100分）

| 维度 | 权重 | 计算方式 | 满分条件 |
|------|------|---------|---------|
| **换手率历史分位** | 30% | `to_r` 的 10年历史分位数（`pe_ttm.y10.mcw.cvpos` 同理计算换手分位） | 换手率分位 > 90% = 100分 |
| **成交额占全市场比例** | 25% | 行业 `ta` / 全A日成交额（通常2-4万亿） | 占比高于历史75分位 = 100分 |
| **估值历史分位** | 25% | PE 10年历史分位数（越高越拥挤） | PE分位 > 85% = 100分 |
| **资金集中度变化** | 20% | 北向/融资净流入持续性（连续3日净流入为高集中） | 连续5日净流入且加速 = 100分 |

**综合拥挤度评分**：
```
crowding_score = 换手分位子分 × 30% + 成交占比子分 × 25% + 估值分位子分 × 25% + 资金集中度子分 × 20%
```

**降级版（缺资金集中度数据时）**：
```
crowding_score_simple = 换手分位子分 × 40% + 成交占比子分 × 35% + 估值分位子分 × 25%
```

### 第四步：风险等级判断

| 分数段 | 风险等级 | 含义 | 操作建议 |
|-------|---------|------|---------|
| 0-39 | 低（GREEN） | 无明显拥挤迹象 | 可正常参与 |
| 40-59 | 中（YELLOW） | 开始出现拥挤信号 | 控制仓位上限 |
| 60-79 | 高（ORANGE） | 拥挤度显著，资金竞争激烈 | 不建议加仓，关注止盈 |
| 80-89 | 极高（RED） | 历史高拥挤区域，多数指标告警 | 建议减仓 |
| 90-100 | 危险（DARKRED） | 历史极少见的极端拥挤 | 建议清仓或回避 |

### 第五步：脆弱触发器识别（五类）

识别以下五类脆弱触发器，逐一判断是否触发：

#### 触发器 1：量价背离
```
定义：成交量（成交额）近5日萎缩 > 25%，但价格仍维持在高位（近5日最大回撤 < 5%）
信号含义：接盘意愿减弱，价格虚高，随时可能补跌
告警级别：HIGH
```

#### 触发器 2：成交萎缩
```
定义：行业/板块日成交额连续3日低于近20日均值的 70%
信号含义：资金活跃度显著下降，主力或开始撤退
告警级别：MEDIUM
```

#### 触发器 3：龙头破位
```
定义：板块龙头股（市值最大的前2家）收盘价跌破 20日移动均线
信号含义：龙头股领先反映资金撤退意愿，板块行情可能逆转
告警级别：HIGH
```

#### 触发器 4：涨停封板率下降（概念板块）
```
定义：近3日涨停封板率（实际封板数/触板数）< 60%，且较峰值下降 > 20ppt
信号含义：主力资金在高位撤退，散户竞争资金不足以维持封板
告警级别：HIGH（仅概念板块适用）
```

#### 触发器 5：北向/融资资金突然净流出
```
定义：连续2日北向资金净卖出且融资余额持续下降
信号含义：外资或杠杆资金开始撤出，行情脆弱性增加
告警级别：MEDIUM（数据不可得时跳过）
```

**触发器判断结果**：
- 0个触发：脆弱性低
- 1-2个触发（含 MEDIUM）：脆弱性中等，建议减仓至正常仓位
- 1个以上 HIGH 触发：脆弱性高，建议清仓或回避
- 3+个触发：脆弱性极高，立即回避

### 第六步：操作建议生成

综合拥挤度评分 + 脆弱触发器，生成可操作建议：

```
最终建议 = f(crowding_score, triggered_count, has_high_trigger)

建议规则：
- score < 60 AND triggered = 0  → "可正常参与"
- 60 ≤ score < 80 AND triggered ≤ 1  → "控制仓位，不加仓，关注止盈时机"
- score ≥ 80 OR has_high_trigger  → "建议减仓，当前板块拥挤度已处于历史高位"
- score ≥ 90 OR triggered ≥ 3  → "建议清仓或回避，多重风险信号同时触发"
```

### 第七步：输出结构化结论

按 `references/output-template.md` 输出，必须包含：

1. **拥挤度仪表盘**：总分 + 四维子指标分值（含风险等级颜色标识）
2. **脆弱触发器清单**：已触发 / 临近触发 / 未触发（附具体数值）
3. **操作建议**（可正常参与/控仓观察/减仓/清仓回避）+ 具体仓位建议
4. **失效条件**（使风险评估解除的具体可观测事件）
5. **监控指标**（后续5-10个交易日需关注的指标与阈值）
6. **置信度声明**（缺数时降级，标注影响）

---

## 降级策略

| 场景 | 降级方案 | 置信度影响 |
|------|---------|---------|
| 无资金集中度/北向数据 | 使用三维简版评分（换手40%+成交35%+估值25%），跳过触发器5 | -10% |
| 无历史换手率分位 | 用绝对换手率阈值（>3%高换手）替代分位数判断 | -15% |
| 概念板块无封板率数据 | 跳过触发器4，其余四类触发器正常评估 | -10% |
| 龙头股价格数据不可得 | 跳过触发器3（龙头破位），标注缺口 | -10% |

**降级不静默：所有降级必须在 `data_gaps` 中显式标注。**

## 与其他 skill 的协同接口

- **← concept-board-analyzer**：当概念处于"拥挤期"时，自动调用本 skill 获取完整拥挤度评分，结果回传给 concept-board-analyzer 更新参与建议
- **← industry-board-analyzer**：高综合评分的行业，在进一步操作前先经本 skill 评估拥挤度
- **→ research-conclusion.schema.json**：拥挤度评分作为 `risks[]` 和 `invalidation_conditions[]` 的重要来源

## 重要注意事项

- **拥挤度高 ≠ 必跌**：高拥挤度表明脆弱性增加，但不等于一定下跌（催化剂持续时可维持拥挤）；关键是结合脆弱触发器综合判断
- **A股情绪极端化**：A股在高拥挤度后的快速下跌往往幅度较大（10-30%），T+1制度下流动性急剧恶化，风险管理要求更高
- **概念板块拥挤度比行业板块更不稳定**：概念板块的拥挤度峰值往往只持续1-5个交易日，需要更高频的监控
- **本 skill 是风险刹车，不是主动策略**：用于评估进场/持仓风险，不用于生成做空信号

数据获取详见 `references/data-queries.md`，指标计算详见 `references/methodology.md`。
