# 概念板块主题生命周期分析器

扮演主题投研分析师。通过"生命周期阶段识别 + 主题热度评分 + 拥挤度风险判断"三层框架，判断概念板块当前所处阶段，给出明确的参与时机建议。

---

## 工作流程

### 第一步：确认输入参数

确认以下内容：
1. **概念名称**：目标概念板块（如"AI算力"、"人形机器人"、"低空经济"）
2. **观察窗口**：回溯期，默认近1个月
3. **分析深度**：全量（生命周期+热度+拥挤度）/ lifecycle（仅阶段判断）/ heat（仅热度评分）/ risk（仅拥挤度）

### 第二步：数据获取

从 `references/data-queries.md` 获取：
- 概念板块成员股列表与近期涨跌幅
- 涨停家数时序（日维度，近10个交易日）
- 换手率5日均值及其历史分位数
- 资金净流入（可用 AKShare `stock_fund_flow_concept`）
- 核心龙头股的量价数据

若数据部分缺失，在 `data_gaps` 中显式标注，降级方式见"降级策略"章节。

### 第三步：生命周期阶段识别（四阶段框架）

| 阶段 | 核心特征 | 信号指标 | 参与建议 |
|------|---------|---------|---------|
| **启动期** | 首板龙头出现，涨停家数≤3，概念知名度低 | 首板龙头出现 + 连续2日涨停扩散 + 资金净流入转正 | 积极关注，轻仓试探 |
| **扩散期** | 涨停家数持续增加（3→10+），跟风资金持续涌入 | 涨停家数≥5 + 换手率环比上升30%+ + 资金净流入持续3日 | 可参与，控制仓位 |
| **拥挤期** | 换手率突破历史80分位，新进资金边际减少，龙头开始出现分歧 | 换手率分位>80% + 涨停封板率下降 + 高位量价背离 | 控仓观察，准备降仓 |
| **退潮期** | 龙头高位滞涨，涨停板封板率持续下降，资金净流出 | 龙头收盘价连续2日低开 + 封板率<50% + 资金净流出 | 回避或清仓 |

**阶段判断规则**：
- 优先看**连续性**（3日以上的信号一致性），单日信号不足以定阶段
- 若多个指标阶段判断矛盾（如涨停数多但资金净流出），判定为"阶段切换期"，降级输出为 WARNING

### 第四步：主题热度评分（0-100分）

综合以下四维指标计算热度评分：

| 维度 | 权重 | 计算方式 | 满分标准 |
|------|------|---------|---------|
| 涨停家数强度 | 35% | 近3日日均涨停数 / 历史峰值 | ≥ 10家 = 100 |
| 换手率加速度 | 30% | 近5日换手率环比增速 | 日均+15%以上 = 100 |
| 资金净流入 | 25% | 近3日净流入 / 板块市值 | 净流入≥1% = 100 |
| 新闻/公告热度 | 10% | 相关新闻条数（相对估计） | 主观评估 1-10分×10 |

**热度评分解读**：
- 80-100：主题极热，关注拥挤度风险
- 60-80：主题活跃，可积极参与
- 40-60：主题中温，适合轻仓观察
- 20-40：主题降温，不建议加仓
- <20：主题退潮，回避

### 第五步：拥挤度风险评估

> **触发条件**：若热度评分 > 70 或阶段处于"拥挤期"，必须执行此步骤。
> 可联动 `board-crowding-risk-monitor` skill 获取完整拥挤度评分（推荐）。

简版快速判断（无 board-crowding-risk-monitor 时的降级方案）：

| 拥挤度信号 | 触发阈值 |
|-----------|---------|
| 换手率高位 | 换手率5日均值 > 历史80分位 |
| 量价背离 | 成交量萎缩>30% 但价格仍在高位 |
| 龙头破位 | 板块龙头股收盘价跌破20日均线 |
| 封板率下降 | 涨停封板率连续3日 < 60% |

出现≥2个信号：高拥挤度，建议减仓或回避。

### 第六步：输出结构化结论

按 `references/output-template.md` 输出，必须包含：

1. **阶段判断**：当前生命周期阶段 + 主要依据 + 阶段持续时间估计
2. **热度评分**：0-100分 + 四维子评分
3. **拥挤度风险**：低/中/高/极高 + 主要风险信号
4. **参与建议**：进入 / 持有 / 减仓 / 回避
5. **失效触发条件**：使结论反转的具体可观测事件
6. **监控清单**：未来5-10个交易日需关注的具体指标与阈值
7. **置信度声明**：基于数据完整性的置信度（0-1）

## 降级策略

| 缺数场景 | 降级方案 | 置信度影响 |
|---------|---------|---------|
| 无资金流数据（AKShare 不可用） | 仅用涨停数+换手率两维评分，权重重分配 | 置信度 -15% |
| 无历史换手率分位 | 用绝对换手率（>3%高换手）替代分位判断 | 置信度 -10% |
| 成员股列表不完整 | 用Top10成员股代表板块，标注覆盖比例 | 置信度 -10% |
| 单日数据（无连续性） | 降级为"信号参考"，阶段判断需用户补充 | 置信度 -20% |

**所有降级必须在输出的 `data_gaps` 字段中显式标注，不允许无声略过。**

## 与其他 skill 的协同接口

- **→ board-crowding-risk-monitor**：当阶段进入"拥挤期"或热度评分 > 70 时，自动触发获取完整拥挤度评分；将结果并入本 skill 的输出
- **← limit-up-down-linkage-detector**：通过联动路径识别哪些概念板块处于扩散期，为阶段判断提供联动依据
- **← sector-rotation-detector**：行业轮动切换时，判断主题概念与当前行业配置方向的一致性

## § 独立调用接口

本 skill 支持被外部插件（`valuation`、`deep-research`、`regime-lab`）独立加载，无需通过 Orchestrator。

**最小输入**：
```
{
  "concept_name": "AI算力",           // 概念板块名称
  "as_of_date": "2026-03-27",        // 分析基准日期
  "mode": "lifecycle"                // lifecycle | heat | risk | full
}
```

**标准输出字段**：
```
{
  "lifecycle_stage": "EXPANSION",    // STARTUP | EXPANSION | CROWDED | DECLINING
  "heat_score": 75,                  // 0-100
  "crowding_risk": "MEDIUM",         // LOW | MEDIUM | HIGH | EXTREME
  "recommendation": "HOLD",         // ENTER | HOLD | REDUCE | AVOID
  "confidence": 0.72,               // 0-1
  "data_gap_note": "..."            // 有缺口时必填
}
```

**加载指令**（供外部插件使用）：
```
Load skill: .claude/plugins/industry-concept-research/skills/concept-board-analyzer/SKILL.md
```

## 重要注意事项

- **阶段 ≠ 绝对可买/可卖**：阶段判断是时机参考，不是绝对操作指令
- **概念 vs 行业本质区别**：概念是主题/事件驱动，行业是基本面驱动；概念的持续性通常弱于行业，退潮更快
- **慎用单日涨停数判断**：单日涨停数受市场整体情绪影响大，需看连续性（≥3日）才有统计意义
- **A股特性**：T+1制度下，拥挤期龙头股流动性可能急剧下降，高位买入风险极高
- 本 skill 输出仅供研究参考，不构成投资建议

数据获取详见 `references/data-queries.md`，指标口径详见 `references/methodology.md`。
