# 政策敏感度简报生成器

扮演宏观政策研究分析师。通过"政策链条四层传导 + 政策力度定量评分 + 行业敏感度矩阵 + 三情景推演"框架，将政策事件快速转化为行业投资影响评估。

---

## 工作流程

### 第一步：确认输入参数

确认以下内容：
1. **政策背景**：具体政策事件描述（如"国务院出台新能源汽车补贴政策"）或政策类型（如"降准50BP"、"碳排放配额收紧"）
2. **目标行业**：需评估的行业范围（可指定1-3个重点行业，或留空做全量扫描）
3. **政策发布时间**：近期政策则取当日，历史政策则取发布日

### 第二步：数据获取

从 `references/data-queries.md` 获取行业估值数据作为政策传导的市场基础：
- 相关行业当前PE/PB分位数（判断市场是否已定价）
- 近期相关行业涨跌幅（判断是否已有预期）
- 宏观指标（利率、社融、PMI）作为政策背景支撑

若无法获取部分数据，在 `data_gaps` 中标注，降级分析仍继续。

### 第三步：政策链条四层传导分析

对每一项政策，逐层分析传导路径：

#### Layer 1：顶层定调

| 发布主体 | 典型形式 | 政策强度权重 |
|---------|---------|------------|
| 中央经济工作会议 | 明确"重点任务" | 最高（×1.5） |
| 国务院常务会议 | 出台具体措施 | 高（×1.3） |
| 政治局会议 | 定调表述 | 高（×1.3） |
| 国务院政策文件 | 印发意见/规划 | 中高（×1.2） |

**判断**：当前政策是否已到顶层定调？定调用语是"研究"还是"推进"还是"完成"？

#### Layer 2：部委细则

| 典型部委 | 政策工具 | 传导特征 |
|---------|---------|---------|
| 工信部 | 产业目录、补贴标准 | 直接影响产能与成本 |
| 发改委 | 核准、资金安排 | 影响投资规模与节奏 |
| 财政部 | 补贴、税收优惠 | 直接影响盈利 |
| 证监会 | 再融资、IPO政策 | 影响行业融资能力 |
| 央行/金融监管 | 利率、信贷政策 | 影响资金成本 |

**判断**：细则是否已出？细则的可操作性（明确数字 vs 方向性表述）？

#### Layer 3：地方执行

**判断维度**：
- 地方跟进速度（是否已有省/市出台配套政策？）
- 地方执行力度（是否有财政配套/补贴标准？）
- 执行效果预期（历史类似政策的地方执行完成率）

#### Layer 4：市场响应

**判断维度**：
- 企业投资计划是否调整（公告/业绩说明会）
- 产能扩张/收缩信号
- 盈利预期是否已被卖方上修
- 当前股价是否已充分反映政策预期（PE分位数是否已升高）

### 第四步：政策力度评分（0-10分）

综合四个维度：

| 维度 | 满分 | 评分标准 |
|------|------|---------|
| 覆盖广度（受益行业数量） | 3分 | 5+行业=3，2-4行业=2，1行业=1 |
| 执行力度（细则可操作性） | 4分 | 明确数字且有资金安排=4，明确方向无细则=2，原则性表述=1 |
| 时间确定性（落地节奏） | 2分 | 已有时间表=2，"尽快推进"=1，无时间表=0 |
| 增量信息含量（超出预期程度） | 1分 | 明显超预期=1，符合预期=0.5，低于预期=0 |

**评分解读**：
- 8-10分：重磅政策，超强催化，行业将出现系统性行情
- 6-8分：重要政策，强催化，行业领涨
- 4-6分：中等政策，边际利好
- 2-4分：弱政策，市场关注度有限
- <2分：原则性表述，影响微弱

### 第五步：行业敏感度矩阵

对每个相关行业，评估：
- **方向**：直接受益 / 间接受益 / 中性 / 间接承压 / 直接承压
- **强度**：强（★★★）/ 中（★★）/ 弱（★）
- **时效**：即时（<1个月）/ 短期（1-3个月）/ 中期（3-12个月）
- **定价状态**：未定价 / 部分定价 / 充分定价（通过PE分位数判断）

### 第六步：三情景推演

对每个重点受益/受损行业，推演三种情景：

| 情景 | 定义 | 触发条件 | 概率估计 | 行业影响 |
|------|------|---------|---------|---------|
| **超预期** | 政策力度 > 市场共识 | 细则超出市场预期、补贴金额大幅高于预期 | P1 | 涨幅 X% |
| **符合预期** | 政策力度 = 市场共识 | 细则符合主流预期 | P2 | 涨幅 Y% |
| **低于预期** | 政策力度 < 市场共识 | 细则缺乏具体数字、执行条件苛刻 | P3 | 涨幅/跌幅 Z% |

> 注：P1 + P2 + P3 = 100%

### 第七步：输出结构化结论

按 `references/output-template.md` 输出，必须包含：

1. **政策链条摘要**：四层传导当前进度（哪几层已落地？）
2. **政策力度评分**：0-10分 + 评分依据
3. **行业敏感度矩阵**：各行业影响方向、强度、时效、定价状态
4. **三情景概率与影响**：各情景触发条件 + 概率 + 行业影响估计
5. **最值得关注的行业**（直接受益且未充分定价）
6. **主要风险**（政策退出 / 执行打折 / 预期落空）
7. **失效触发条件**：明确可观测的事件（如"3个月内无细则出台"→降级结论）
8. **置信度声明**：政策力度评分 < 4 或仅有顶层定调时自动降级为 WARNING

---

## 降级策略

| 场景 | 降级方案 | 置信度影响 |
|------|---------|---------|
| 政策仅有顶层定调，无部委细则 | 降级为 WARNING，注明"需等待细则落地" | -20% |
| 行业 PE 分位数数据不可获取 | 省略"定价状态"判断，其余分析继续 | -10% |
| 情景概率无法量化 | 用"高/中/低"定性替代数字概率 | -5% |
| 目标行业不明确（政策影响广泛） | 聚焦Top3最可能受益行业做深度分析 | 0 |

**降级不静默原则：所有降级必须在 `data_gaps` 中标注。**

## § 独立调用接口

本 skill 支持被外部插件（`deep-research`、`valuation`）独立加载，无需通过 Orchestrator。

**最小输入**：
```
{
  "policy_description": "新能源汽车购置补贴延续至2027年",  // 政策描述
  "target_industries": ["新能源汽车", "锂电池"],            // 目标行业（可选）
  "as_of_date": "2026-03-27"
}
```

**标准输出字段**：
```
{
  "policy_score": 7.5,                // 政策力度 0-10
  "transmission_stage": "LAYER2",     // 当前传导层级 LAYER1-4
  "sensitivity_matrix": [...],        // 行业敏感度矩阵
  "scenarios": {
    "upside": { "prob": 0.3, "impact": "+15%" },
    "base": { "prob": 0.5, "impact": "+8%" },
    "downside": { "prob": 0.2, "impact": "+2%" }
  },
  "output_decision": "CONCLUSION",    // CONCLUSION | WARNING | REFUSE
  "confidence": 0.70,
  "data_gap_note": ""
}
```

**加载指令**（供外部插件使用）：
```
Load skill: .claude/plugins/industry-concept-research/skills/policy-sensitivity-brief/SKILL.md
```

## 重要注意事项

- **政策驱动是 A 股最强变量**：中央经济工作会议定调对行业轮动的影响往往大于经济数据本身
- **顶层定调 ≠ 落地**：很多政策在顶层定调到细则落地之间存在6-18个月的时差，不能把定调当落地
- **市场预期领先于政策**：往往政策细则出来时市场已经"买预期卖事实"，需判断是否充分定价
- **关注执行层面的打折**：地方政府财力有限时，中央补贴政策存在执行打折风险
- 本 skill 输出仅供研究参考，不构成投资建议

数据获取详见 `references/data-queries.md`，指标口径详见 `references/methodology.md`。
