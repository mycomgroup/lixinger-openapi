# DECISIONS — sector-factor-attributor

## 设计决策记录

### D1：为什么用 PE/市值变化近似估值扩张，而不是更精确的 DDM 模型

**背景**：理论上可以用股息折现模型（DDM）或 EV/EBITDA 变化做更精确归因。

**决策**：使用 PE 变化近似估值扩张贡献。

**理由**：
1. A股行业层面缺乏高质量的盈利预期和 FCF 数据，DDM 模型数据可得性低
2. PE 变化是市场参与者最普遍使用的估值度量，与 A 股实际定价逻辑最贴近
3. 近似误差在分析精度要求范围内（±5%），对结论影响有限
4. 可获取性高：理杏仁 API 直接提供历史 PE 数据

---

### D2：为什么设计三档归因质量（FULL/PARTIAL/INDICATIVE）而不是简单的"完整/不完整"

**背景**：可以用二元的"有数据/没数据"标记。

**决策**：使用三档归因质量评级。

**理由**：
1. 不同层级的数据缺失对结论可靠性的影响程度不同，二元标记过于粗糙
2. FULL：三维数据齐全，可直接用于配置决策
3. PARTIAL：缺盈利数据，估值和风险溢价归因仍可靠，配置逻辑仍可参考
4. INDICATIVE：多维缺数，结论仅供方向性参考，不建议直接用于配置
5. 三档与 qc-rules.schema.json 中的 CONCLUSION/WARNING/REFUSE 输出决策逻辑对应

---

### D3：为什么风格因子评估是定性的而不是定量的（如 Barra 风格因子载荷）

**背景**：成熟的风格因子分析需要 Barra 模型或类似的因子收益率数据库。

**决策**：使用基于公开指标的定性/半定量评估。

**理由**：
1. Barra 风格因子载荷需要商业数据库（万得、聚宽等），超出当前数据层支持范围
2. 基于 PE 分位、市值分布、股息率等公开指标的定性评估，虽然不精确，但在大多数场景下足够做配置方向判断
3. 定性评估门槛低，用户更容易理解和验证
4. 如需精确载荷数据，可接入外部量化数据源（作为未来升级方向）

---

### D4：为什么 A股特殊注意中单独强调"政策行政估值"

**背景**：传统估值归因假设 PE 变化纯粹反映市场情绪。

**决策**：在输出中单独标注政策驱动的"行政估值"情形。

**理由**：
1. 2023年开始的"中特估"主题导致国企/央企估值修复，PE扩张不是市场情绪，而是政策引导
2. 如果把政策行政估值归入"情绪驱动"，会错误判断持续性（政策驱动的估值修复持续性可能更强）
3. 识别这种特殊情形有助于用户正确解读因子归因结果
