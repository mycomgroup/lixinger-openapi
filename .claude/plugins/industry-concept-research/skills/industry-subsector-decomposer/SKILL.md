# 行业细分拆解器

扮演行业深度研究分析师。通过"细分层级拆解 + 强弱分层 + 贡献度分解 + Alpha来源识别"框架，将宽泛的行业判断精炼为可操作的细分子方向建议。

---

## 工作流程

### 第一步：确认输入参数

确认以下内容：
1. **目标行业**：申万一级行业名称（如"电子"、"医药生物"、"新能源"）
2. **下钻层级**：`two`（到申万二级，默认）/ `three`（到申万三级，覆盖更细但数据可能不完整）
3. **分析基准日期**：默认今日
4. **强弱参考窗口**：涨跌幅比较期，默认近1M + 近3M

### 第二步：数据获取

从 `references/data-queries.md` 获取：

**步骤 2.1**：获取目标行业的申万二级/三级行业代码列表
```
API: cn/industry, params: {source: "sw", level: "two"|"three"}
过滤：找出属于目标一级行业的细分行业
```

**步骤 2.2**：获取各细分行业基本面数据
```
API: cn/industry/fundamental/sw_2021
params: {date: "{today}", stockCodes: [各细分行业代码], metricsList: ["pe_ttm.mcw","pe_ttm.y10.mcw.cvpos","pb.mcw","mc","to_r","ta"]}
```

**步骤 2.3**：获取各细分行业历史市值用于计算涨跌幅
```
API: cn/industry/fundamental/sw_2021 （startDate/endDate模式，逐个细分行业查询）
计算：近1M / 近3M 涨跌幅 = (当前mc - 历史mc) / 历史mc
```

若部分细分行业数据不可得，在 `data_gaps` 中标注，覆盖比例需告知用户。

### 第三步：细分行业强弱排名

构建排名表：

| 细分行业 | 申万代码 | 近1M涨跌幅 | 近3M涨跌幅 | PE分位 | 综合强弱分 |
|---------|---------|---------|---------|------|---------|
| ... | ... | ... | ... | ... | ... |

**综合强弱分计算**（0-100）：
- 近1M动量贡献：35%（标准化：最强=100，最弱=0）
- 近3M动量贡献：35%（标准化）
- 估值低位加分：15%（PE分位 < 30% 额外加分）
- 换手率活跃度：15%（换手率处于历史高位时加分）

### 第四步：强弱分层（三层划分）

| 层级 | 判断标准 | 含义 |
|------|---------|------|
| **领涨层** | 综合强弱分 Top 30% + 近1M涨跌幅高于一级行业均值 | 当前行情的核心受益细分 |
| **跟涨层** | 综合强弱分中间 40% + 跟随一级行业整体趋势 | 行情扩散时的跟随方向 |
| **掉队层** | 综合强弱分 Bottom 30% + 跑输一级行业 | 行情中被边缘化的细分 |

**分层注意事项**：
- 若细分数量 < 5，分层调整为 领涨/跟涨/掉队 各取 1-2 个
- 若所有细分行业都显著跑输市场，优先输出"行业整体偏弱"结论

### 第五步：贡献度分解

计算各细分行业对一级行业总涨跌幅的贡献：

```
贡献度 = 细分行业涨跌幅 × (细分行业市值 / 一级行业总市值)
```

输出：
- 前三名贡献来源（贡献了一级行业X%的涨跌幅）
- 是否存在结构分化（领涨层贡献度 > 60%）

### 第六步：Alpha 来源识别

判断目标一级行业的行情属于：

| Alpha 来源类型 | 特征 | 含义 |
|-------------|------|------|
| **行业 Beta 驱动** | 所有细分行业协同上涨，分散均匀 | 宏观/行业整体受益，难以通过细分选择获取超额 |
| **细分 Alpha 驱动** | 仅1-2个细分领涨，其余掉队 | 存在结构性机会，需精选细分 |
| **主题概念驱动** | 领涨细分与概念主题高度相关（如AI、出海） | 主题溢价，需关注概念持续性 |

### 第七步：输出优先子方向

基于领涨层 + 估值分位 + Alpha来源，输出 Top 2-3 优先子方向：

每个优先子方向包含：
- 细分行业名称 + 申万代码
- 选择理由（强弱分层 + Alpha来源 + 估值支撑）
- 关注的关键数据点（如新订单数据、开工率等）
- 关联的主要龙头股（2-3家）

### 第八步：输出结构化结论

按 `references/output-template.md` 输出，必须包含：

1. **细分行业排名表**（含涨跌幅、估值分位、综合强弱分）
2. **强弱分层结果**（领涨/跟涨/掉队三层）
3. **贡献度分解**（哪些细分贡献了大部分涨跌幅）
4. **Alpha 来源判断**（Beta / 细分Alpha / 主题驱动）
5. **下一阶段优先子方向**（Top 2-3，含选择理由）
6. **置信度声明**（数据覆盖比例 < 70% 时降级）
7. **数据缺口说明**（如有）

---

## 降级策略

| 场景 | 降级方案 | 置信度影响 |
|------|---------|---------|
| 申万三级数据不完整（部分细分无数据） | 降级到二级分析，标注覆盖比例 | -15% |
| 历史市值数据不可查（无法计算涨跌幅） | 仅用估值分位数做强弱排名 | -20% |
| 细分行业数量 < 3（行业较集中） | 直接输出所有细分，无三层分层 | -10% |

- **← industry-board-analyzer**：接收一级行业分析结果后，做细分下钻
- **→ industry-chain-mapper**：将领涨细分传入，分析其在产业链中的传导位置
- **← sector-rotation-detector**：配合轮动信号，识别轮动目标行业内部哪个细分最先受益

## § 独立调用接口

本 skill 支持被外部插件独立加载，无需通过 Orchestrator。

**最小输入**：
```
{
  "industry_name": "电子",
  "level": "two",             // two | three
  "as_of_date": "2026-03-27"
}
```

**标准输出字段**：
```
{
  "subsector_ranking": [...],       // 细分排名表（数组）
  "strength_tiers": {
    "leading": [...],               // 领涨层细分
    "following": [...],             // 跟涨层细分
    "lagging": [...]                // 掉队层细分
  },
  "alpha_source": "SUBSECTOR_ALPHA",// BETA | SUBSECTOR_ALPHA | THEME_DRIVEN
  "priority_subsectors": [...],     // 优先子方向 Top 2-3
  "confidence": 0.75,
  "data_coverage": 0.85,            // 数据覆盖比例
  "data_gap_note": ""
}
```

**加载指令**：
```
Load skill: .claude/plugins/industry-concept-research/skills/industry-subsector-decomposer/SKILL.md
```

## 重要注意事项

- **细分数据质量差异**：申万三级行业数据覆盖不如二级完整，三级分析结论需谨慎
- **A股细分轮动快**：细分行业内部轮动速度很快（1-2周），短期细分判断有效期有限
- **龙头效应显著**：部分细分行业由1-2家龙头主导，细分涨跌幅主要反映龙头股行情

数据获取详见 `references/data-queries.md`，方法论详见 `references/methodology.md`。
