# 方法论 — industry-subsector-decomposer

本文档定义行业细分拆解的核心算法、强弱分层规则和 Alpha 来源识别框架。

---

## 1. 细分行业强弱排名算法

### 1.1 综合强弱分计算（0-100）

```
综合强弱分 = 近1M动量分 × 35% + 近3M动量分 × 35% + 估值低位加分 × 15% + 换手率活跃度 × 15%
```

**各维度计算方式**：

**近1M/3M动量分（标准化）**：
```python
def normalize_score(perf_list):
    """将涨跌幅列表标准化到 0-100"""
    min_v = min(perf_list)
    max_v = max(perf_list)
    if max_v == min_v:
        return [50] * len(perf_list)
    return [(v - min_v) / (max_v - min_v) * 100 for v in perf_list]
```

**估值低位加分**：
```python
# PE 10年分位数越低，加分越高（低估值 + 动量好 = 双击机会）
pe_score = max(0, 100 - pe_pos * 100)    # pe_pos 为 0-1 的分位数
```

**换手率活跃度**：
```python
# 相对于历史均值的换手率
# 换手率高于历史均值 30% 以上 = 活跃，100分
# 低于均值 = 按比例得分
to_r_score = min(100, (to_r / to_r_avg) * 70)   # 简化近似
```

### 1.2 强弱分层规则

```
领涨层：综合强弱分排名 Top 30% 且近1M涨跌幅高于一级行业中位数
跟涨层：综合强弱分排名 中间 40%
掉队层：综合强弱分排名 Bottom 30% 或近1M涨跌幅明显低于一级行业中位数
```

**边界处理**：
- 细分数量 < 5：领涨层取Top 1-2，跟涨层取中间，掉队层取Bottom 1-2
- 细分全部跑输市场：优先输出"行业整体偏弱"结论，三层分层仅供内部对比参考

## 2. 贡献度分解公式

### 2.1 细分行业对一级行业涨跌幅的贡献

```python
# 一级行业加权涨跌幅
industry_perf = sum([perf_i × (mc_i / total_mc) for each subsector i])

# 细分行业贡献度
contribution_i = perf_i × (mc_i / total_mc)
contribution_pct_i = contribution_i / industry_perf * 100  # 占比（%）
```

**解读阈值**：
- 单个细分行业贡献度 > 60%：高度集中，行业行情由该细分主导
- 前三名合计贡献度 > 80%：结构高度集中，细分选择至关重要
- 贡献度较均匀（前三名合计 < 50%）：行业整体 Beta 驱动，细分选择价值有限

## 3. Alpha 来源识别规则

### 3.1 三种 Alpha 来源类型

**Rule 1：行业 Beta 驱动**
```
条件：全部细分行业近1M涨跌幅 标准差/均值 < 0.5（变异系数低）
     且 前三名贡献度合计 < 50%（分布均匀）
结论：本轮行情是一级行业整体受益，细分内部无明显分化
含义：通过配置一级行业 ETF 已能充分受益，无需细分选择
```

**Rule 2：细分 Alpha 驱动**
```
条件：前三名细分涨幅 > 一级行业均值 × 1.5
     且 部分细分涨幅与一级行业均值差异 > 20百分点
结论：存在结构性机会，需精选细分
含义：在一级行业板块内部，特定细分跑出显著 Alpha，精选 > ETF
```

**Rule 3：主题概念驱动**
```
条件：领涨细分行业名称/龙头股与近期热门概念高度相关（AI/出海/低空等）
     且 领涨细分 PE 分位数已明显高于其他细分（溢价扩大）
结论：主题溢价，需关注概念持续性
含义：主题概念溢价通常持续时间有限，需跟踪催化剂是否持续
```

### 3.2 优先子方向选择逻辑

```
优先选择满足以下条件的细分（按优先级排序）：
1. 属于领涨层（综合强弱分 Top 30%）
2. PE 分位数 < 60%（估值未明显高估，尚有安全边际）
3. 换手率处于历史中等偏高位置（市场关注度在提升）
4. 有明确的基本面改善逻辑支撑（非纯主题驱动）

输出 Top 2-3 个优先子方向，每个附上关键选择理由。
```

## 4. 信号规则（可测试规则）

**Rule A（细分领涨预示行业行情扩散）**：
```
IF {行业内 1-2 个细分近5日涨幅 > 行业均值 × 2 AND 换手率提升 > 50%}
THEN {未来2-4周，行业内其他细分跟涨概率 > 55%}
CONFIDENCE {0.55; 需历史验证}
FAILURE_MODE {概念退潮（资金撤出）; 监管政策打击领涨细分}
```

**Rule B（细分强弱分化信号）**：
```
IF {领涨层与掉队层近1M涨幅差异 > 25百分点 AND 贡献度集中度 > 70%}
THEN {行情属于细分 Alpha 驱动，ETF 策略跑输精选策略概率 > 65%}
CONFIDENCE {0.65; 历史案例支撑}
FAILURE_MODE {风格切换（由细分 Alpha 回归行业 Beta）}
```

## 5. 降级策略详细说明

| 场景 | 影响 | 降级方法 | 置信度扣减 |
|------|------|---------|---------|
| 三级行业数据覆盖 < 70% | 三级分析可靠性低 | 仅输出二级行业分析 | -15% |
| 历史市值数据不全（无法计算涨跌幅） | 无法做动量分析 | 仅用估值分位数排名 | -20% |
| 细分数量 < 3 | 分层意义有限 | 列出所有细分，不做三层分层 | -10% |

---

## 6. 边界条件与失效场景

- **行业整体跑输**：若所有细分都跑输市场，细分选择价值有限，优先输出"行业整体配置价值低"
- **新行业/新细分**：历史数据少于12个月的细分，PE 分位数可靠性低，需注明
- **中特估/国企改革驱动**：部分细分因国企改革主题大涨，非传统基本面驱动，需单独标注
